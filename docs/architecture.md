## Page Generation Pipeline

### Upload Flow (new sticker)
```
User uploads via web form (upload.js)
  -> File stored in Supabase Storage (stickers bucket)
  -> Record inserted in stickers table
  -> EXIF data extracted (GPS, date), reverse geocoded
  -> n8n webhook triggered (async, non-blocking)
    -> Social media post scheduled (if enabled)
    -> GitHub Actions repository_dispatch triggered
      -> generate-single-sticker.js {id}
        -> stickers/{id}.html + clubs/{club_id}.html + countries/{CODE}.html
      -> optimize-images.js --only={id}
        -> {image}_web.webp (600x600, q80) + {image}_thumb.webp (150x150, q75)
      -> generate-city-pages.js
        -> cities/{slug}.html
      -> git commit + push -> deploy
```

## Game Modes

| Mode | Timer | Description |
|------|-------|-------------|
| Classic | 10 sec/question | Guess club, earn points, combo streaks |
| Time To Run | 60 sec total | As many as you can in a minute |
| Daily Quiz | 45 sec start | 18 stickers, daily challenge for everyone |

## Battle Mode

ELO rating system for sticker comparison. Algorithm uses exponential weighting for fair distribution:
- New stickers (games=0) get 40% forced selection chance
- Recent stickers get 80% weight reduction (soft exclusion)
- Three modes: Close rating (40%), Random (40%), Extreme matchups (20%)

3. 302 redirects (vercel.json) are temporary, change to 301 after stabilization.
4. Service Key != anon key. Keep secret.
5. Image optimization takes 30-60 min for all stickers.
6. When changing SEO in templates/generators, update ALL generator scripts consistently.
