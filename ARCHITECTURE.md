# StickerHunt Architecture

## Project Overview
Global football sticker database with 3200+ stickers, 675 clubs, 54 countries, 22+ cities.
Static HTML pages generated by Node.js scripts, hosted on GitHub Pages via Vercel.

## Tech Stack
- **Frontend:** HTML/CSS/JS (static pages)
- **Backend:** Supabase (PostgreSQL + Storage + Auth + Edge Functions)
- **Hosting:** GitHub Pages + Vercel (hybrid)
- **Automation:** n8n webhooks + GitHub Actions
- **Domain:** stickerhunt.club

## Page Generation Pipeline

### Upload Flow (new sticker)
```
User uploads via web form (upload.js)
  → File stored in Supabase Storage (stickers bucket)
  → Record inserted in stickers table
  → EXIF data extracted (GPS, date), reverse geocoded
  → n8n webhook triggered (async, non-blocking)
    → Social media post scheduled (if enabled)
    → GitHub Actions repository_dispatch triggered
      → generate-single-sticker.js {id}
        → stickers/{id}.html (sticker page)
        → clubs/{club_id}.html (club page)
        → countries/{CODE}.html (country page)
      → optimize-images.js --only={id}
        → {image}_web.webp (600x600, q80)
        → {image}_thumb.webp (150x150, q75)
      → generate-city-pages.js
        → cities/{slug}.html
      → git commit + push → deploy
```

### Generators

| Script | Creates | Trigger |
|--------|---------|---------|
| `generate-single-sticker.js` | sticker + club + country pages | Each new sticker upload (GitHub Actions) |
| `regenerate-club-pages.js` | club + country pages | Manual (mass regeneration) |
| `regenerate-country-pages.js` | all 54 country pages | Manual or bulk update |
| `regenerate-club-sticker-pages.js` | sticker pages for one club | Club rename |
| `regenerate-stickers-batch.js` | specific sticker pages by IDs | Manual fix |
| `regenerate-all-stickers.js` | ALL sticker+club+country pages | Major update (rare) |
| `generate-city-pages.js` | all city pages + index | Each sticker upload |
| `optimize-images.js` | _web.webp + _thumb.webp | Each sticker upload |
| `enrich-clubs-ai.js` | wiki-cache.json (AI descriptions) | Manual (one-time) |
| `fetch-wiki-data.js` | wiki-cache.json (Wikipedia data) | Manual |

**⚠️ IMPORTANT:** `generate-single-sticker.js` and `regenerate-club-pages.js` both have their OWN copies of helper functions for club/country page generation (generateWikiSection, generateClubDescription, generateStickerGallery, generateSchemaJsonLd, generateCountryPage, COUNTRY_FLAGS, etc). When changing layout — update BOTH files!

### Image Pipeline

```
Original upload → Supabase Storage: stickers/club_{id}/{timestamp}_{filename}.jpeg
                → Edge Function (async): creates _web.webp + _thumb.webp
                → GitHub Actions: optimize-images.js (backup, same result)

URLs in HTML:
  OG image:  {base}_web.webp (top-rated sticker by ELO)
  Gallery:   {base}_thumb.webp (150x150)
  Detail:    {base}_web.webp (600x600)
  Full size: original .jpeg
```

### GitHub Actions Concurrency

- `concurrency: { group: generate-pages, cancel-in-progress: false }` — serializes runs
- Only ONE pending run is queued; additional concurrent triggers are **cancelled**
- When batch-uploading stickers, trigger one at a time with ~45s delay between them
- Push retry with `git stash` → `git pull --rebase` → `git stash pop` (up to 3 attempts)
- **Self-healing:** each run for sticker N regenerates sticker N-1's page (navigation links)
